Why Claude Code

Why Code, not Chat or Cowork?

No preview overhead
Reads sheets programmatically. No file cards, no "reading file…" UI. Values in, synthesis out.
Composability
Chain tools: read a sheet → write a CSV → run a script → ask a follow-up. Chat can't do that.
Persistence
Local folders, files, project memory, git. Chat has Projects; Cowork has workspaces. Neither matches a filesystem.

One-off question? Chat. Shared team workflow? Cowork. Solo recurring analysis? Code.

A Fair Question

Could I do this in Cowork?

Yes — Cowork has the same Google Workspace connector. The answers would be similar.

Cowork is better if…
  • • Multiple people on your team use the same analysis
  • • You don't want to install anything on your computer
Code is better if…
  • • You're running this personally, for yourself
  • • You want to combine sheet data with local files or scripts

I picked Code because this is solo and recurring.
If your version is team-shared, Cowork is the right answer.

How to Set This Up

Two ways to connect Google Sheets

The easy path
Built-in Google Workspace connector
Three clicks. Uses your Google login. Right default for most people.
The advanced path
Service account + gws CLI
More setup. File-by-file scope. For personal data or team deployments.

Let's look at each.

Path 1 — The Easy Path

Built-in Google Workspace connector

  • Open Claude Code settings → Connectors
  • Click "Connect Google Workspace"
  • Authorize in Google's consent screen

30 seconds. Uses your personal Google login. The right default for most people.

Claude sees whatever your Google account sees — all of Drive, optionally Gmail and Calendar if you enable those scopes.

Path 2 — The Advanced Path

Service account + gws CLI

When you want more control:

  • ●File-by-file scope — Claude only sees sheets you explicitly share
  • ●Personal Gmail and Calendar off-limits by construction
  • ●Demonstrable data-minimization story for compliance or audit

Responsible Use

Know what your plan allows

  • ●A DPA is required to process personal data. Anthropic provides a Data Processing Agreement on Team and Enterprise plans. Personal/Pro: no DPA.
  • ●Data minimization still applies. Even with a DPA, only send what you actually need. Service-account scoping helps you prove it.
  • ●Check training opt-out. Team and Enterprise: off by default. Verify before connecting sensitive data.

A DPA is the legal floor.
Scoping and minimization are the practical floor.

Responsible Use

What not to send to Claude

By default, keep personal data out. If you absolutely need to send it, anonymize it first.

Keep out (or anonymize if essential)
  • • Full customer or employee names
  • • Email addresses
  • • Phone numbers
  • • Postal addresses
  • • Personal financial details tied to named individuals
  • • Health or sensitive data (GDPR special category — keep out entirely)

You don't need names to answer "how much ARR is at risk."
Send the shape, not the identities.
